1. Executive Summary
Reflct e una web app AI-powered per il virtual try-on di abbigliamento in tempo reale, pensata
come app mobile-first pubblicabile su iOS e Android (tramite PWA o wrapper nativo). L'utente
punta la fotocamera su se stesso e vede qualsiasi capo da qualsiasi e-commerce sovrapposto al
proprio corpo mentre si muove — fronte e retro.
Il prodotto si distingue per tre pilastri: tecnologia AR avanzata (OOTDiffusion + MediaPipe), un
sistema di guardaroba digitale persistente ispirato a Pinterest, e un social layer virale (Outfit
Battle, Rate My Outfit, AI Stylist). Il design visivo adotta un'estetica scrapbook/collage Y2K —
sfondo crema, font misti, elementi con bordi solidi e ombre inkstyle — distinto da qualsiasi altra
app di moda sul mercato.
Nome Reflct
Dominio reflct.app (disponibile)
Tipologia Web App PWA + App mobile (iOS/Android) — SaaS B2C con
evoluzione B2B
Design system Scrapbook / Collage / Y2K — sfondo crema, font misti, bordi inkstyle
Core feature AR try-on real-time (fronte/retro), scraping e-commerce, body scan,
size advisor, guardaroba, social, price tracker
Stack frontend Angular 17 (PWA) + Capacitor per iOS/Android
Stack backend Spring Boot 3 (Java 21) + Python FastAPI (AI) + Playwright (scraping)
Database / Storage PostgreSQL 16 + Redis + MinIO (S3-compatible)
Pagamenti Stripe + PayPal + Apple Pay + Google Pay
Modello business B2C free+premium (7,99/14,99 euro/mese) > API/Shopify > B2B
embed > Affiliate

2. Design System — Estetica Scrapbook
2.1 Direzione Visiva
L'estetica di Reflct e ispirata alle immagini di riferimento fornite: francobolli vintage internazionali,
lettere ritagliate da riviste (ransom note / Y2K), stelle patch ricamate, fotocamere analogiche,
collage su carta crema. L'obiettivo e un'app che sembra un mood board fisico — calda, giocosa,
personale — contro la freddezza delle app di moda convenzionali.
Il principio guida e il contrasto intenzionale tra il contenuto tech (AR, AI, scraping) e la forma
visiva analogica. Questo crea memorabilita e differenziazione immediata.

2.4 Regole Bordi e Ombre
• Bordi: sempre 2px solid #2A2420 (ink) su tutti gli elementi interattivi e card
• Ombra inkstyle: box-shadow: 3px 3px 0 #2A2420 — nessuna ombra sfumata, solo offset
solido
• Border-radius: 12px per card standard, 100px per pill/badge/CTA, 4-8px per elementi
piccoli
• Rotazione leggera: alcune card tilted a -1deg o +1deg per effetto collage
• Bordi tratteggiati (dashed): per elementi ghost, divider, pannelli secondari
• Tape: striscia rgba(240,200,100,0.45) in cima a card e sidebar — effetto scotch
2.5 Comportamento Hover / Interazione
• Card: hover => transform translate(-1px,-2px) + box-shadow 4px 4px 0 ink
• Bottone principale: hover => translate(-1px,-1px) + shadow piu grande
• Social buttons: hover => rotate(-3deg) scale(1.05) — comportamento sticker
• Wardrobe card: hover => rotate(1deg) scale(1.02)
• Navbar icon: hover => rotate(0deg) scale(1.1) da -5deg di default

4. Architettura Tecnica
4.1 App Mobile-First — Angular + Capacitor
Reflct nasce come Angular 17 PWA (Progressive Web App) utilizzabile da browser mobile. Per la
pubblicazione su App Store e Google Play si usa Capacitor, che wrappa la webapp Angular in un
container nativo senza dover riscrivere il codice. Questo approccio permette di sviluppare una
volta sola e distribuire su tre piattaforme (web, iOS, Android).
Piattaforma Approccio
Web (browser) Angular 17 PWA — installabile da browser come app
iOS (App Store) Capacitor wrapper — build nativa da Angular, pubblicata su App Store
Android (Play Store) Capacitor wrapper — build nativa da Angular, pubblicata su Google Play
4.2 Stack Completo
Layer Tecnologie
Frontend / App Angular 17 (PWA), TypeScript, Capacitor (iOS/Android), PrimeNG, SCSS,
Three.js, TensorFlow.js, WebRTC, MediaRecorder API
Backend Spring Boot 3, Java 21, Spring Security (JWT), Spring Data JPA, WebSocket
(STOMP), REST API versionate
AI Service Python 3.11, FastAPI, MediaPipe, OOTDiffusion, BodyMeasure Net, TFLite,
OpenCV, rembg, GPT-4o-mini (stylist)
Scraping Service Python 3.11, Playwright (headless Chromium), BeautifulSoup4, LLM extractor
fallback
Database PostgreSQL 16 (dati), Redis 7 (cache/sessioni/pub-sub), Flyway (migration)
Storage MinIO (S3-compatible): foto utente, body scan, video try-on, immagini capi
Pagamenti Stripe (carta/Apple Pay/Google Pay), PayPal — nessun dato carta
memorizzato su server Reflct
DevOps Docker Compose, GitHub Actions (CI/CD), Nginx (reverse proxy + HTTPS),
Let's Encrypt

5. Feature Set Completo
5.1 AR Try-On Real-Time
Feature Dettaglio tecnico
Try-on live webcam (fronte) TensorFlow.js BlazePose + Three.js overlay a ~25fps lato client
Rilevamento rotazione Angolo spalle MediaPipe: 0 gradi = fronte, 180 gradi = retro, soglia
automatico 45 gradi
Try-on retro in tempo reale Modello OOTDiffusion separato addestrato su viste posteriori
Rendering AI alta qualita Frame sampling ogni 500ms verso AI Service, output 512x512 in
circa 800ms (GPU)
Registrazione video MP4 MediaRecorder API, export verticale 9:16 ottimizzato per Reels e
TikTok
Retro sintetico ControlNet genera vista posteriore se sito ha solo foto frontale
5.2 Import E-Commerce & Scraping
• Incolla URL di qualsiasi sito — Playwright headless estrae foto, misure, prezzo, brand
• Parser dedicati per top 20 siti: Zara, H&M, ASOS, Zalando, Mango, Shein, Amazon
Fashion, Farfetch
• Fallback LLM (GPT-4o-mini) per siti non mappati
• Estrazione retro automatica dalla gallery del prodotto
• Snapshot prezzo al momento del salvataggio + monitoraggio periodico
• Strategia di uscita: partnership API ufficiali Zalando, Amazon Product Advertising,
Shopify plugin
5.3 Body Scan & Size Advisor
• Scansione guidata alla registrazione: foto frontale + laterale OPPURE video rotante 360
gradi
• Calibrazione tramite altezza inserita manualmente dall'utente
• BodyMeasure Net stima 7 misure: busto, vita, fianchi, spalle, torso, inseam, manica
(precisione circa 2-3 cm)
• Size Advisor: confronto misure corpo vs. tabella taglie del capo estratta dallo scraping
• Output: taglia consigliata evidenziata con score fit (verde/giallo/rosso) e testo esplicativo
generativo
• Aggiornamento misure disponibile in qualsiasi momento dal profilo
5.4 Guardaroba & Retention
• Tutti i capi provati salvati automaticamente con foto/video, dati capo e taglia consigliata
• Categorizzazione automatica AI (CNN su DeepFashion): vestiti, top, pantaloni, giacche,
scarpe, accessori
• Correzione categoria con drag & drop + categorie personalizzate
• Outfit Builder: combina capi del guardaroba in outfit completi
--- PAGE 9 ---
• Capsule wardrobe tematiche (es. look da lavoro, look estate 2025)
• Storico prezzi con sparkline 90 giorni, badge minimo storico
• Price Alert: soglia personalizzabile con notifica push/email al ribasso
• Notifiche AI contestuali: cambio stagione, riscansione corpo, look virali
5.5 Social Layer
• Rate My Outfit: pubblica look nella feed pubblica, voti emoji (fire/meh/skip) in tempo reale
• Outfit Battle: sfida A vs B votata dalla community in 24h, video side-by-side 9:16 per
TikTok
• AI Stylist: GPT-4o-mini analizza guardaroba e suggerisce outfit completi con mood e
descrizione
• Feed trending: look piu votati della settimana, visibile senza account
5.6 Condivisione
• Link permanente: pagina pubblica sempre accessibile con preview look, dati capo, link
e-commerce
• Link temporaneo: scadenza a scelta 24h/7gg/30gg
• Link protetto: PIN a 4 cifre opzionale
• Share nativo: WhatsApp, Instagram Stories, TikTok, X — export verticale 9:16 con
overlay branded
• Pagina pubblica senza account: CTA iscrizione, no paywall, no tracking aggressivo

6.2 Periodo di Prova
• Premium: 30 giorni gratuiti — NESSUNA carta richiesta per iniziare
• Al termine dei 30 giorni: reminder 7gg e 1gg prima della scadenza
• Se non si inserisce carta entro la scadenza: downgrade automatico a Free (nessun
addebito)
• Conversione attesa da trial a paid: target 15-25% (benchmark SaaS consumer)
6.3 Metodi di Pagamento
• Carta di credito/debito — via Stripe
• PayPal — via Stripe PayPal integration
• Apple Pay — via Stripe (iOS Safari e app nativa)
• Google Pay — via Stripe (Android Chrome e app nativa)
• Sicurezza: dati carta MAI memorizzati sui server Reflct, tutto tokenizzato via Stripe
6.4 Revenue Tiers (priorita)
• Priorita 1 — B2C free/premium/pro: revenue ricorrente, scala con base utenti
• Priorita 2 — API try-on + Shopify plugin: 0,05 euro/rendering pay-per-use, 49 euro/mese
plugin
• Priorita 3 — B2B widget white-label: 199/599/custom euro/mese per brand
• Priorita 4 — Affiliate commission: 5-12% su acquisti tracciati via link UTM
